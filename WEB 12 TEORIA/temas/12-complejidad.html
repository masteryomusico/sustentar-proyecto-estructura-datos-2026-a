Definición y características

  • La notación Big O describe el límite superior del crecimiento de un algoritmo en el peor caso.
  • Las complejidades más comunes son: O(1) constante, O(log n) logarítmica, O(n) lineal, O(n log n), O(n²) cuadrática.
  • La complejidad temporal mide el número de operaciones; la espacial mide la memoria usada.
  • Al diseñar algoritmos siempre buscamos reducir la complejidad al mínimo posible según el problema.

Ejemplo 01

O(n) – Búsqueda lineal

def buscar(lista, objetivo):
    for elemento in lista:  # O(n)
        if elemento == objetivo:
            return True
    return False
Recorremos cada elemento una sola vez. Si la lista tiene n elementos, en el peor caso hacemos n comparaciones.
Ejemplo 02

O(log n) – Búsqueda binaria

def binaria(lista, objetivo):
    ini, fin = 0, len(lista) - 1
    while ini <= fin:       # O(log n)
        mid = (ini + fin) // 2
        if lista[mid] == objetivo: return mid
        elif lista[mid] < objetivo: ini = mid + 1
        else: fin = mid - 1
    return -1
Cada iteración descarta la mitad del espacio de búsqueda. Para 1 millón de elementos, son solo ~20 comparaciones.
